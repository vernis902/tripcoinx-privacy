Privacy Policy

Last updated: May 28, 2026

Overview

This Privacy Policy describes how TripCoinX ("the app", "we") handles information when you use our Android application. TripCoinX is a simple offline currency calculator for travelers.

Data We Collect

TripCoinX stores or transmits only the following categories of data, either directly or through the service providers described below:

  • Currency settings — your selected source and target currencies and the exchange rate you have entered manually. Stored locally on your device only using Android SharedPreferences. Never transmitted to any server.
  • Anonymous crash diagnostics — if Firebase Crashlytics is enabled for a given release, TripCoinX may automatically send anonymous technical diagnostics to help us monitor app stability. This may include crash stack traces, app version, Android version, selected currency pair, and limited diagnostic breadcrumbs around events such as onboarding completion, currency pair change, conversion attempts, rate refresh outcomes, or purchase flow milestones. We do not send typed amounts, feedback message content, payment card details, contacts, or precise location data through Crashlytics. Firebase may assign a pseudonymous installation identifier for crash grouping. TripCoinX does not currently enable standalone Firebase Analytics collection in production. If that changes in a future release, we will introduce it with an explicit in-app choice and updated privacy disclosures.
  • Feedback message and technical diagnostics — when you voluntarily submit a bug report or feature request through the in-app Help & Feedback screen, the text of your message is sent together with the following technical data to help resolve issues faster:
    • App version name and code
    • Android API level
    • Selected currency pair (source → target)
    • Active theme (system / light / dark)
    • Device language tag (e.g. pl-PL)
    • Network type at submission time (WiFi / Cellular / Offline)
    • A randomly generated anonymous UUID (stored locally, never linked to your identity)

    This data is transmitted only when you explicitly tap "Send" in the feedback form. The technical diagnostic payload does not include your name, email address, device model, precise location, or other direct identifiers. However, any information you choose to type into the feedback message itself will be transmitted as part of that report, so please avoid including sensitive personal information unless it is necessary for your request. The payload is processed by Make.com and used to create a private issue in the developer's bug tracker. It is not sold, shared with third parties, or used for advertising.

  • Ad-related technical data — when banner ads are shown in the free version, the Google Mobile Ads SDK (AdMob) may automatically collect and share technical data needed to serve, measure, and protect ads. According to Google's Mobile Ads SDK disclosure, this may include your IP address, app and device identifiers, basic device and app diagnostics, and product interaction events such as app launches, ad impressions, or taps. At the time of this policy, TripCoinX removes the Android AD_ID permission from its manifest, so the app build does not intentionally request the Android Advertising ID. Google may still process other identifiers, such as App Set ID or similar technical identifiers, as part of ad delivery, analytics, or fraud prevention.

Third-Party Services

TripCoinX uses the following third-party services:

  • Google AdMob / Google Mobile Ads SDK — displays banner ads in the free version of the app. To serve ads, limit ad frequency, and prevent fraud, this SDK utilizes device local storage and technical identifiers that function similarly to website cookies. When an ad request is made, Google may receive data such as IP address, app and device identifiers, interaction data, and diagnostics for advertising, analytics, and fraud prevention. At the time of this policy, TripCoinX removes the AD_ID permission from its manifest, meaning the standard Android Advertising ID is not collected; however, essential local caching and diagnostic tracking are still utilized by the SDK infrastructure as required for operation. See Google Mobile Ads SDK data disclosure and Google's Privacy Policy.
  • frankfurter.app — provides exchange rate data sourced from the European Central Bank (ECB). When the app refreshes rates in the background, it sends a standard HTTP GET request to api.frankfurter.app. The request contains no user data — only the base currency code (e.g. EUR). No API key or account is required. frankfurter.app is an open-source, public service; its source is available at github.com/lineofflight/frankfurter.
  • Google Play Billing Library — processes in-app purchases. When you make a purchase, Google processes the transaction on its own infrastructure. We do not receive or store your payment details. See Google's Privacy Policy.
  • Google Firebase Crashlytics — used to monitor crashes and collect anonymous technical diagnostics described above. Firebase is operated by Google. Firebase Analytics is not enabled in production at the time of this policy. If enabled in a future release, it will be introduced separately with updated disclosures. See Firebase Privacy and Security and Google's Privacy Policy.
  • Make.com (Integromat) — receives feedback reports submitted through the Help & Feedback feature. Data includes the message content you choose to send together with the accompanying technical diagnostics described above. Make.com's privacy notice is available at make.com/en/privacy-notice.

Data Sharing

We do not sell personal information. The third-party transmissions associated with normal app use are:

  • Standard HTTP requests to frankfurter.app for exchange rate updates — contains no user data.
  • Ad requests and related technical data to Google AdMob / Google Mobile Ads SDK when ads are displayed — may include IP address, app and device identifiers, interaction data, and diagnostics as described above.
  • Anonymous crash diagnostics to Google Firebase Crashlytics — used only for app stability monitoring.
  • Feedback message content and accompanying diagnostic data submitted through the Help & Feedback form — only when you explicitly tap "Send", only to Make.com, as described above.

We do not sell or trade personal information. Crash diagnostics and feedback submissions are not shared for advertising purposes. Ad-related technical data may be shared with Google AdMob for ad delivery, measurement, analytics, and fraud prevention as described above.

Data Retention and Deletion

All local app data (currency settings and exchange rate) is stored only on your device. You can delete it at any time by:

  • Going to Settings → Apps → TripCoinX → Clear Data, or
  • Uninstalling the app

Data transmitted to third-party services such as Google AdMob, Google Play Billing, Firebase Crashlytics, or Make.com is handled on those providers' systems and retained according to their own privacy policies and retention practices.

Children's Privacy

TripCoinX does not knowingly collect any data from children under the age of 13.

Changes to This Policy

We may update this Privacy Policy from time to time. The "Last updated" date at the top of this page will reflect any changes. Continued use of the app after changes constitutes acceptance of the updated policy.
